Tag `inputs` allow users to create multiple tags within a single input field, with each tag displayed as a separate item. This feature is often used for entering things like keywords or categories.

## Component Usage
### Default

`Tag inputs` display individual tags within the input field, followed by a text area for entering additional tags.

<STagInput client:only="vue" />
<Code code={STagInputRaw} lang="vue" title="default.vue" />

### Required

A `required` tag input ensures that users must fill in a value before submitting a form. If left empty, the form will prompt an error, preventing submission until the field is completed.

<STagInputRequired client:only="vue" />
<Code code={STagInputRequiredRaw} lang="vue" title="required.vue" />

### Custom Validator

A tag input with a `custom validator` allows for validation rules to be defined manually, ensuring that input meets specific conditions. It provides flexibility to enforce custom logic during form submission.

<STagInputCustomValidator client:only="vue" />

### Custom Type

A `custom type` tag input allows for tags to be displayed in specific colors. This feature enhances visual organization by assigning different colors to tags based on their type or category.

<STagInputCustomType client:only="vue" />
<Code code={STagInputCustomTypeRaw} lang="vue" title="customType.vue" />

### Custom Tag

A `custom tag` input allows users to create and define unique tags based on their preferences. This flexibility enables personalization and better organization of information within the input field.
